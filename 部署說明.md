# 數辦設備借用清單 — 部署說明

新版架構:**靜態前端(index.html)+ 精簡 GAS API(Code.gs)+ Google 試算表存資料**。

## 為什麼會比原版快很多

| 原版(GAS 整頁網頁) | 新版 |
|---|---|
| 每次開頁都要等 Google 沙箱 iframe 載入,2~5 秒起跳 | 前端是純靜態 HTML,零外部資源,毫秒級開啟 |
| 每個動作透過 `google.script.run` 來回 1~3 秒 | 開頁先顯示上次快照,背景再同步;操作只打一次 API |
| 每次都即時讀試算表 | 伺服器端快取 30 秒 + 批次讀取,多人同開只讀一次 |

## 部署步驟(約 10 分鐘,只需做一次)

### 第 1 步:建立試算表與後端

1. 到 [sheets.new](https://sheets.new) 建立新試算表,命名如「數辦設備借用清單」。
2. 選單「擴充功能 → Apps Script」,刪掉編輯器裡的預設內容,貼上 `Code.gs` 全部內容,存檔。
3. 上方函式下拉選 **`setup`** → 按「執行」→ 依提示授權(選你的帳號 → 進階 → 前往專案)。
   完成後試算表會多出「設備」「紀錄」兩張工作表,並附 12 項範例設備。
4. 右上「部署 → 新增部署作業」:
   - 類型:**網頁應用程式**
   - 執行身分:**我**
   - 誰可以存取:**任何人**(必選,否則前端讀不到;資料本身仍只在你的試算表裡)
5. 複製產生的網址(結尾是 `/exec`)。

### 第 2 步:設定前端

打開 `index.html`,找到開頭的這一行,貼上剛才的網址:

```js
const API_URL = "https://script.google.com/macros/s/……/exec";
```

> 留空 = 示範模式,資料只存在本機瀏覽器,適合先試用畫面。

### 第 3 步:放上網(擇一)

- **GitHub Pages(推薦,免費)**:建一個 repo,把 `index.html` 傳上去,Settings → Pages 開啟,得到固定網址發給全校。
- **Netlify Drop**:到 [app.netlify.com/drop](https://app.netlify.com/drop) 把檔案拖進去就有網址。
- **最陽春**:直接把 `index.html` 用 email/雲端硬碟發給同仁,**雙擊打開也能用**(仍會連到同一份試算表)。

## v3 升級:正式設備匯入 + 盤點機制(2026-07-17)

已部署過 v2 的話,照以下順序做一次:

1. **更新後端**:Apps Script 編輯器貼上新版 `Code.gs` 全部內容,存檔。
2. **匯入正式設備**:函式下拉選 **`importEquipment`** → 執行。會把「設備」工作表換成
   同仁「設備借用動態總表」的 46 項正式設備(270 個序號),並:
   - 清掉示範設備與測試紀錄,寫入總表上目前已借出的 29 件(22 筆借用中紀錄);
   - 建立「盤點」工作表,故障(8 件)與長期借用(4 件)的序號登記為停借;
   - 保留總表既有的 18 筆 iPad 盤點紀錄(2026-04-23)。
   ⚠️ 重複執行會把三張工作表重設回總表狀態,之後的正式資料請勿再執行。
3. **重新部署**:「部署 → 管理部署作業 → 編輯 → 版本選『新版本』」(網址不變)。
4. 前端 `index.html` push 到 GitHub 後自動更新,不用再動。

### 盤點怎麼用

- 網頁新分頁「**盤點**」列出每項設備的**上次盤點日**,超過 90 天(或從未盤點)標「待盤點」。
- 按「盤點」逐序號勾選 **正常/異常/未尋獲/長期借用**;借出中的序號自動略過。
- 非「正常」的序號立刻**停借**(借用視窗選不到),修好後再盤一次改回「正常」即恢復。
- 每筆結果寫入「盤點」工作表(日期、序號、結果、盤點人),完整歷程可回溯。

## 日常維護

- **新增/修改設備**:直接編輯試算表「設備」工作表(編號、名稱、分類、總數、備註、序號清單),30 秒內全校頁面自動更新,不用改任何程式。
- **序號管理**:「序號清單」欄留空時,系統自動以「編號-01、編號-02…」依總數產生序號;若有正式財產編號,直接填入該欄(逗號分隔,例如 `PC1130001, PC1130002`),數量以填入的序號數為準。借用時選序號、歸還時可只勾選部分序號。
- **借用與歸還**:全部在網頁上操作,自動寫進「紀錄」工作表。
- **改了 Code.gs 之後**:要「部署 → 管理部署作業 → 編輯 → 版本選『新版本』」才會生效,網址不變。

## 常見問題

- **開頁顯示「連線失敗」**:多半是部署時「誰可以存取」沒選「任何人」,重新部署一次即可。
- **想限制只有校內能用**:GAS 端把存取權改「網域內任何人」即可,但前端 fetch 會被登入頁擋住——這種情況建議改用「優化版 GAS 單一網頁」方案,再跟 Claude 說一聲就能改。
- **紀錄太多會變慢嗎**:API 只回傳借用中 + 最近 100 筆已歸還紀錄,試算表本身留完整歷史,不影響速度。
